# UI / UX Rules

## Quality bar
Use the strongest existing Nadif premium surfaces as the benchmark.

## Principles
- operational clarity
- strong hierarchy
- low clutter
- fast scanning
- explicit actions
- calm colors and spacing
- no dead-end states

## Required for every surface
- loading state
- empty state
- error state
- success feedback
- disabled state when relevant
- mobile consideration if user-facing

## Avoid
- cramped admin tables by default
- random cards everywhere
- duplicate buttons
- hidden critical actions
- placeholder labels
- unexplained statuses

## Preferred patterns
- command-center style layouts for complex staff tasks
- step-by-step flows for multi-stage actions
- detail drawers or side panels for secondary editing
- consistent status chips and badges
