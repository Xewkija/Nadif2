# Repo Structure

## Recommended layout

- `src/app` or `src/pages`
- `src/components`
- `src/features`
- `src/hooks`
- `src/lib`
- `src/types`
- `src/state`
- `src/contexts`
- `supabase/migrations`
- `supabase/functions`
- `docs`
- `.github`

## Structure rules
- group by domain first when possible
- keep backend contracts near domain APIs
- avoid large generic dumping grounds
- avoid deeply nested folders without clear value
